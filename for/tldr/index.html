AndrewOS is 24 Claude Skills, packaged as an installable plugin. Each one carries frontmatter that pins its scope, its tools, and its model tier (opus / sonnet / haiku). That's the catalog you're asking someone to start.
"autonomous AI agents for sales, marketing, and operations"
My agents run autonomous LinkedIn content production, prospecting, content distribution, and scheduled retrospective loops across two businesses. Not a prototype. My actual daily operations.
"non-technical teammates building workflows independently"
The system is folder-based and spec-first so the operator gives a goal, not code. I productize this exact thing as a $5K "OS Builder" for other small-business owners.
"composable systems where AI tool outputs feed into other tools"
Arena-based delegation with validation chains. career-ops, one of my skill suites, pipes a job scan into scoring into a report into a tailored CV, end to end, with no hand-holding between steps.
"systems that know when to escalate"
Every arena has human gates, and a builder / reviewer / validator triad where the validator grades the builder and blocks on a regression before anything ships. Escalation is a design principle, not an afterthought.

# one of ~24 skills in AndrewOS — frontmatter pins context + cost
name: aos-retrospective
model: opus # judgment work earns the premium tier

The cheap skills (content drafting, scheduling) run on haiku; the judgment skills run on opus. That cost instinct is in the frontmatter, not bolted on after. It's the same discipline as the two-tier model router I built into FieldCommand.
