I built a two-tier model router for FieldCommand that sends cheap work to a cheap model and harder work to a stronger one, with three providers wired in behind one interface and per-tenant feature flags gating each AI feature. That is a proxy with a paved road on top. I run it in production now.

Veritas is an evaluation workflow I shipped. It reads an AI claim, retrieves the source, and returns a structured verdict with a confidence cap. I drove its false-positive rate down by reading failures one at a time, which is exactly what an eval loop is for.
"Design and implement systems that improve the performance, reliability, safety, and cost efficiency of AI-powered experiences"
Cost efficiency is why the router exists, and reliability is why AndrewOS runs builder, reviewer, and validator agents where the validator grades the builder and blocks bad output before it ships. I treat cost, latency, and safety as the design, not an afterthought.
"You will work mostly in TypeScript & Python. Comfort with typed languages and modern backend practices is a must"
FieldCommand is a multi-tenant SaaS I built solo in TypeScript on a PostgreSQL and Supabase backend, with REST integrations, ETL pipelines, role-based access control, and tenant isolation. Typed backend work is my daily ground.
"Build tools that help other engineers move faster, simplifying complex workflows and improving how teams build and ship software"
AndrewOS is a developer platform for agents. I built reusable delegation patterns, structured task packets, and validation gates so that other agents can pick up work and ship it safely. The whole point is leverage for the people downstream of me.
