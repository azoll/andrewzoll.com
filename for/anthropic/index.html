Built for the Applied AI Architect, Partnerships role

A reference architecture a partner can reuse: route to the right model, then gate the output with code.

The architect's job is to codify a working Claude pattern into something a partner can repeat, prove, and sell. This is that pattern, running live. A request routes across Anthropic, OpenAI, and Google, then a deterministic eval and guardrail step decides whether the output ships. It fails closed, the same way my Veritas verifier did when it dragged false positives from about 30 percent to under 5 percent.

Live demo. Real provider calls through the Vercel AI Gateway. The model answers and self-reports signals. Pure code renders every verdict.

Run a request to see the router pick a provider and the eval gate render a pass, review, or reject. Try the "PII leak" or "no citation" scenario to watch it fail closed.

The model never renders the verdict. It answers the request and self-reports signals about its own answer. A pure function in lib/anthropic-router-core.mjs turns those signals into pass, review, or reject, which is why the same signals always produce the same verdict and every verdict traces to a rule. That core runs as offline unit tests, the same code path as production. This is the design choice a partner needs to trust the pattern in their own deal.
