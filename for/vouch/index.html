This is the thing I build. COI Check reads an insurance certificate with a model, then a pure rules engine renders the pass or fail. I wrapped it as an MCP server so an agent calls check_coi, gets a Zod-typed result, and the verdict is computed, never generated. The model reads, code decides.

I run an agent operating system across two businesses: builder, reviewer, and validator triads where a validator grades the builder and blocks bad output before it ships, plus a determinism layer where code computes and verifies while the agent only extracts and judges.
